Algorithm - Depth First Search

基本介紹

深度優先搜尋演算法（Depth-First-Search，DFS）是一種用於遍歷、搜尋樹或圖的演算法。DFS會以「先看到先搜尋」的方式，儘可能深地搜尋樹(or圖)的分支。當某個目標成功被找到，或是當前節點v所有的邊都己被探尋過，演算法將回溯到上一個節點，並持續重複「搜尋&rarr;找到目標or沒路可走&rarr;回溯」的遞迴形式，直到最後要馬成功找到目標，要馬所有的節點都被我們遍歷完為止(i.e. 用來暫存遞迴的system stack為空)。

解題觀念

  1. 確認圖的形式：在開始前我們必須先釐清我們要traverse的東西是什麼，以下列舉幾個常見的形式。
    • Tree：題目會給root，資料結構通常為TreeNode，traverse的方式通常為利用遞迴函式往left-most的方向下探
    • Graph：常見為以下兩種
      • Matrix(two-dimension array)：資料結構為一個二維陣列，traverse的方式通常使用double for loops作為框架，而遞迴函式作為實際上的search。
      • Adjancency list：資料結構為一個dictionary，keys代表每一個node(or node value)，values代表其對應的key的所有adjancent nodes，即Dict[str: List]。
  2. 建立遞迴函式：為最關鍵的一步，主要可分為三個部分
    • 終止條件：
      • 「達成某個目標」：此時要和下方第三點呼應，開始思考要「return什麼」，e.g.若題目要求找到某個字串或多個數組，那找到後要用哪種形式儲存起來？視題目而定。
      • 「已經沒路可走了」：這就要看圖的形式為何，e.g.若為leetcode常見的matrix，那條件可能就是「row and column」不能超過二維陣列的長度(即current cell必須為一個有效的cell)。
      • Note: 這邊的「沒路可走」不僅包含判斷當前有無有效的cell可以走，也包含判斷某個路是否已經visited，幾乎所有圖的traversal為了避免任一個節點被重複遍歷到，都會用某種形式去將紀錄走過的路，常見方法有
        1. label法：將訪問過的cell用某個特殊的char標記(e.g.”#”)，此時視題目情況，看要不要在回溯時還原labeled cells原本的值。
        2. 查表法：將訪問過的cell透過某種資料結構(e.g.set, list)暫存下來，這邊一樣視題目情況，看要不要在回溯時進入表內刪除visited cell的值。
    • 遞迴遍歷的方向：
      • 若圖為matrix：通常為「上、下、左、右」，順序不限。
      • 若圖為adjancency list：沒有特定方向，看該node的adjancent nodes有誰(即該node在dictionary內所對應的list)。
    • 實際回傳的東西：
      實際寫作時可以試著去思考，對於某個節點而言，若已經無路可走(或說所有的鄰居都被我們訪問過)，則此時有需要做哪些post processing嗎？(i.e.有沒有什麼資料是應該被我們手動回復到之前的狀態)，而做完處理後，回傳那些「應該被保存起來並一路回傳回去的」資訊。
