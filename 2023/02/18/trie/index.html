Data Structures - Trie

基本介紹

Trie 是一種樹狀結構也稱為字典樹，用以在大量字串中快速找尋是否存在該字串。此外，Trie也有人叫前綴樹(Prefix Tree)，因為它是由字串間相同的前綴文字組成。
一個典型的Trie如下：[car], [cat], [dot]

觀念

  1. Root只作為一個起始點，不存任何東西
  2. 任一Node可以有多個children，數量不限
  3. 擁有相同前綴字的word會順著同一個方向往下長(e.g. “fuck”, “fucker”的overlapping nodes為”f”, “u”, “c”, “k”)
  4. Trie是由TrieNode組成，而TrieNode主要儲存兩樣東西：
    • is_word : 即current char是否為某個字的結尾，以boolean表示且初始化為False
    • child：即current char的所有子節點，以dictionary表示(keys, values分別為「用來索引到某個子節點的char」以及「可以被索引到的子節點」)

操作

常見的operation主要包含：

  1. Insert(word): 將”word”插入Trie內 &rarr; 為word的iteration，從root開始，利用O(1)查找char是否存在於子節點，若不存在則新建一個TrieNode，並將char新增到current node的children裡(i.e. 將char當作key、TrieNode當作value assign到dictionary內)，反之則順著char移動到該node，重複上述動作直到迴圈結束，此時ptr指向的node必為word的最後一個char，所以記得將該node改為True，表示其為word的結尾。
  2. Search(word)：查看word是否存在於Trie &rarr; 為word的iteration，從root開始，利用O(1)查找char是否存在於子節點，若不存在就直接return False，反之則移動到該node並繼續檢查，若迴圈結束表示Trie確實存在word中的這些char，但注意：並不表示這些char在Trie為一個word，e.g. Insert(“pencil”), Search(“pen”)為False，因為雖然”p”, “e”, “n”存在於Trie，但”pen”在Trie中不是一個單字，所以迴圈結束後還要檢查ptr現所指的node是否為True。
  3. StartFrom(prefix): 查看Trie中是否存在前綴為”prefix”的word &rarr; 為word的iteration，從root開始，利用O(1)查找char是否存在於子節點，若不存在就直接return False，反之則移動到該node並繼續檢查，若迴圈結束了，代表prefix確實存在，所以return True
